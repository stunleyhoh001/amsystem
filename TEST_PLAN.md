# 简单POS测试计划

## 1. 基础打开

- 使用 `http://localhost` 或 Firebase Hosting 地址打开系统。
- 顶部版本应显示最新版本。
- 断网后刷新，页面仍可打开。

## 2. 收银流程

- 使用已授权邮箱登录 POS。
- 登录后右下角应显示当前班次开始时间。
- 选择当前分行。
- 点击 `简单草本减脂计划第一阶段` 加入购物车。
- 实收金额应自动预填为应收金额。
- 输入折扣后，应收和找零应重新计算。
- 选择付款方式并填写参考号，小票和转账记录应显示付款资料。
- 选择 `简单支付 / SimplePay` 后，小票、转账记录和付款汇总应显示该方式。
- 非现金付款未填写参考号时，应出现提醒确认。
- 完成收款后，应出现小票。
- 今日销售应出现订单，可用客户、电话或订单号搜索。
- 管理员在转账记录作废订单时，需要输入 `VOID`；完成后订单应显示已作废，库存应回补。
- 转账记录超过 8 笔时，`显示更多` 可展开记录。
- 转账记录可按付款方式筛选。
- 转账记录页的当天结算汇总应跟随日期和付款方式筛选变化。
- 转账记录页可导出当天结算 CSV，内容应包含汇总和明细。
- 结束班次后，设置页应出现交班记录，可导出交班 CSV。

## 3. 后台管理员

- 使用本机配置的管理员邮箱进入后台。
- 日期范围切换为今天、本月、全部，确认全局指标变化。
- 分行表现、热销商品、每日销售趋势应跟随日期范围变化。
- 付款方式汇总应跟随日期范围变化。
- 付款汇总 CSV 可以下载。
- 客户跟进可标记 `已联系` 和 `已完成`。
- 已完成客户不应继续显示在待跟进列表。
- 导出客户名单，确认包含计划结束日期、到期状态、跟进状态和累计消费。

## 4. 分行和授权

- 新增一个测试分行。
- 授权一个测试 POS 用户到该分行。
- 切换到新分行后，库存应独立显示。

## 5. 库存和报表

- 调整商品库存，库存流水应新增记录。
- 作废订单后，库存流水应出现订单作废回补记录。
- 菜单页和库存页搜索商品名、SKU 或分类，应能过滤列表。
- 修改业务设置里的低库存提醒阈值，低库存列表应跟着变化。
- 导出销售记录、分行汇总、付款汇总、商品排行、库存总表、客户名单、审计日志、库存流水。
- CSV 用 Excel 打开时，中文应正常显示。

## 6. 离线同步

- 登录后断网。
- 完成一笔收款，顶部应显示待同步。
- 修改客户跟进状态，顶部应显示订单更新待同步。
- 恢复网络后点击补传离线订单，待同步数量应下降。

## 7. 数据保护

- 导出完整 JSON 备份。
- 从备份恢复前应出现确认。
- 清空全部数据时，必须输入 `RESET` 才会继续。

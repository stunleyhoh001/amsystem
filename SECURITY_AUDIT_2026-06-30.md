# 三系统代码审计记录

审计日期：2026-06-30

审计范围：

- 简单POS
- 一分钟支付 / SimplePay
- 一分钟联盟营销系统

## 已修复

1. POS 开班、现金冲正和交班改为保存成功后才更新画面状态。
2. POS 分行、员工、设置、商品和本机重置增加输入校验及失败保护。
3. 重复 SKU 不再静默覆盖旧商品，非数字售价不能保存。
4. 员工不能手工修改跨系统关联资料；此操作改为管理员专用。
5. POS 整合云函数会重新核对任务 ID、来源、目标、分行、订单状态、金额、付款参考号、联盟电话、推荐码和配套。
6. 任务取消后，旧的处理中结果不能覆盖取消状态。
7. SimplePay 付款前会再次核对 POS 任务类型、订单号、分行和金额。
8. SimplePay 商家可结算余额不足时不再允许自动退款，避免凭空增加顾客积分。
9. 联盟系统兼容 `customerPhone` 和 `customer.phone`，修复正常订单无法匹配用户的问题。
10. 联盟任务处理期间若 POS 同时作废，会自动撤销联盟权益或进入人工复核。
11. 联盟云函数内会写入数据库的乱码中文已修正。
12. 新离线密码改用 PBKDF2-SHA256 及 210,000 次派生，旧 SHA-256 密码记录继续兼容。
13. POS 交易列表商品名称增加 HTML 转义。

## 上线阻挡项

1. SimplePay 的 `secureMoneyFunctionsEnabled` 仍为 `false`。此状态只适合迁移和测试，不能作为正式资金安全模式。
2. 最新 Firestore Rules 与三个项目的云函数尚未部署，当前测试只验证本机代码，不能代表线上规则已经生效。
3. 跨项目 IAM 尚未配置，付款、退款、联盟入单和撤销尚未完成真实端到端测试。
5. Firebase Rules 尚未通过 Emulator 或线上 Rules Playground 执行规则级测试。

## 已完成验证

- 三个项目 JavaScript 语法检查通过。
- 简单POS全部自动测试通过。
- SimplePay 安全契约测试通过。
- 联盟 POS 整合契约测试通过。
- SimplePay 与联盟前端动态 HTML 已统一经过清洗器，并新增 XSS 渲染契约测试。
- PBKDF2 新密码与旧密码兼容测试通过。
- 三个项目未发现服务账号私钥或 `BEGIN PRIVATE KEY` 内容。
- 简单POS本地 HTTP 页面及 `v0.88` 脚本读取正常。

## 结论

简单POS本机核心流程已达到继续实机验收的状态，但三系统不能在上线阻挡项完成前宣称正式安全上线。

# 简单POS

当前应用版本：`v0.94`

这是一款适合新手继续扩展的网页 POS。它的使用策略是“联网优先、断网兜底”：平时上线联网使用，浏览器会优先读取服务器最新版；遇到断网时，系统会自动使用本机缓存继续打开和收银。

收银操作采用“本机即时完成、云端后台同步”：即使联网速度慢，完成收款也会先保存到本机并立刻出小票，云端订单和库存随后自动补传。

## 当前预设商品

- 简单联盟 RM180 配套，SKU：`AFF-PLAN-RM180`
- 3 星期跟进服务，售价 RM0

旧演示用 RM150 草本计划已经删除，不会再恢复或同步到云端。

下一步演示时可以填写客户资料：

- 客户姓名
- 电话 / WhatsApp
- 联盟推荐码（选填）
- 联盟配套必须填写客户电话以匹配联盟账号，推荐码仍为选填
- 开班、现金冲正、交班、分行、员工、商品和本机重置均先安全保存，失败时不改变当前资料
- 新设置的员工及管理员离线密码使用 PBKDF2 多轮派生；旧密码记录仍可兼容登录
- 跨系统任务会在云函数端重新核对POS订单、分行、金额、参考号和客户资料
- SimplePay 退款及联盟撤销结果会回写POS，区分已退款、已撤销、需人工复核与退款异常
- Google 管理员可按需从联盟项目同步配套价格；POS不使用持续监听，联盟入单仍做最终价格核对
- Google 管理员可按需检查最近50条整合任务，并只对允许状态执行服务器核验后的安全重试
- Google 管理员可按需执行三系统只读连接诊断，检查跨项目权限、SimplePay安全资金模式、分行商家映射及联盟有效配套
- Google 管理员可在全局总览按 POS 订单号只读追踪三边状态，核对整合任务、SimplePay付款/退款及联盟入单/撤销

完成收款后，小票会保存客户资料。导出 CSV 时，这些资料也会一起导出，方便后续跟进。

## 后台管理员

系统设置了唯一后台管理员，完整邮箱不会显示在页面。

```text
已在本机配置，页面不会公开显示完整邮箱
```

联网时必须使用该管理员的 Google 账号；断网时必须使用管理员邮箱和预先设置的离线密码，才能使用：

- 全局销售总览
- 分行表现汇总
- 开设新分行
- 授权 POS 使用者
- 商品管理
- 恢复示例商品
- 导出销售记录
- 清空全部数据，需输入 `RESET` 二次确认

管理员登录由 Firebase Google Authentication 和 Firestore 用户授权共同验证。离线密码只保存加盐哈希；关闭浏览器会清除登录身份，但不会清除订单、班次或库存。

## 在线和离线策略

建议正式使用时采用这个模式：

- 正常情况：联网使用，读取服务器最新版系统和云端数据
- 断网情况：继续用本机缓存收银，订单先保存在本机
- 恢复网络：把离线订单同步回云端总后台

当前版本已经完成第一步：系统文件“联网优先、断网可打开”。下一阶段要做的是“订单和库存云端同步”，这样总管理员才能看到所有分行的实时数据。

当前 Firebase 已经接入基础层：Google 登录、云端状态、销售记录写入 Firestore、分行和授权用户写入 Firestore、离线订单队列、收款后库存同步。在线收款会优先使用 Firestore transaction 同时写订单和扣库存，降低多设备同时销售时的库存冲突。

如果在线 transaction 因云端库存不足或权限问题失败，本机订单和库存会自动回滚，不会显示成功小票。

管理员后台已经加入 `初始化云端数据`。第一次完成 Firebase 设置后，管理员 Google 登录，再点击这个按钮，就会把本地分行、授权用户、商品上传到 Firestore。

系统也加入了 `刷新云端资料`，可手动从 Firestore 读取分行、授权用户、商品、销售记录、库存流水和审计日志，并保存成本机缓存。

后台现在也会显示：

- 同步状态：还有多少离线订单待补传
- 热销商品：按当前日期范围统计商品销量和销售额
- 每日销售趋势：按当前日期范围查看每日订单数和销售额
- 客户跟进：即将到期或刚到期的计划客户
- 低库存提醒：各分行库存小于或等于 5 的商品
- 业务设置：品牌名、小票标题、默认计划、服务天数、低库存提醒阈值、小票结尾
- 完整备份：导出分行、授权用户、商品、销售、待同步订单和业务设置
- 离线补传：订单和库存变更都会进入待同步队列
- 库存调整流水：记录商品、分行、调整前后库存、调整人和原因
- 审计日志：记录新增分行、授权用户、移除用户、业务设置、备份恢复、清空数据、云端初始化
- 审计和库存报表：管理员可导出 CSV，方便对账、追踪人为调整和留档
- 商品和库存报表：管理员可导出商品销售排行和各分行库存总表
- 客户跟进：管理员可标记已联系或已完成，销售 CSV 会包含跟进状态
- 离线跟进：客户跟进状态使用独立订单更新队列，恢复网络后只更新订单资料，不重复扣库存
- 备份恢复：先恢复到本机，确认正确后再手动初始化云端数据

离线订单队列工作方式：

- 断网或云端不可用时完成收款，订单会先保存在本机待同步队列。
- 顶部云端状态会显示待同步数量。
- 恢复网络并登录云端后，系统会自动补传待同步订单。

Firebase 配置安全：

- `firebase-config.example.js` 是模板，可以提交到 GitHub。
- `firebase-config.local.js` 放真实 Firebase 配置，只留本机，不提交。
- 前台页面不会公开显示完整管理员邮箱。
- `app.js` 只保存管理员邮箱哈希，不保存明文邮箱。
- 如果 GitHub 提醒 Google API Key 泄露，请到 Google Cloud 限制 API Key 的可用域名。

## 本地运行

如果电脑装了 Python，在这个文件夹运行：

```powershell
python -m http.server 4173
```

然后打开：

```text
http://localhost:4173
```

也可以直接双击 `index.html` 预览，但离线安装功能需要通过 `http://localhost` 或 HTTPS 网站访问。

## 上线方式

最简单的方式是把这些文件上传到任意静态网站服务，例如 Netlify、Vercel、Cloudflare Pages、GitHub Pages，或你的服务器静态目录。

上线时至少需要上传：

- `index.html`
- `styles.css`
- `app.js`
- `sw.js`
- `manifest.webmanifest`
- `icon.svg`

注意：正式上线必须使用 HTTPS，浏览器才会启用 PWA 缓存。当前缓存策略是联网优先，断网时才使用离线缓存。

## 数据保存在哪里

当前版本的数据保存在使用者这台设备的浏览器 `localStorage` 中，所以它天生支持离线，但不同电脑之间不会自动同步。

后续如果要做多门店、多设备同步，可以继续增加：

- 登录账号
- 云端数据库
- 库存同步
- 订单同步
- 权限管理
- 会员系统
- 扫码枪适配

## 蓝牙打印机

1. 使用 Android 或电脑的 Chrome / Edge，并通过 HTTPS 网站或 localhost 打开 POS。
2. 到“设置 > 蓝牙小票打印机”选择 58mm 或 80mm。
3. 点击“配对 / 更换”，选择已开机的 BLE ESC/POS 热敏打印机。
4. 先执行“测试打印”，确认后可开启“收款后自动打印”。

打印机配对资料只保存在当前收银设备，不会上传 Firebase。iPhone / iPad 浏览器和只支持传统 Bluetooth Classic SPP 的打印机无法使用网页蓝牙直连，此时请使用收据页的“系统打印”。

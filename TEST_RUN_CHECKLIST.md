# 简单POS逐步测试记录

测试建议顺序：先测本机基础功能，再测 Firebase 云端，最后测离线同步。每完成一项就在前面打勾。

## 0. 测试前准备

- [ ] 使用 Chrome 或 Edge 打开系统。
- [ ] 打开页面后，顶部版本显示 `v0.94`。
- [ ] 如果页面仍显示旧版，按 `Ctrl + F5` 强制刷新一次。
- [ ] 确认 Firebase Google 登录已加入授权域名，例如 GitHub Pages 域名和 localhost。
- [ ] 确认 Firestore 规则已经发布。
- [ ] 管理员点击 `检查三系统连接`，SimplePay、分行商家及联盟项目会分别显示连接状态。
- [ ] 尚未配置跨项目 IAM 时，诊断会显示对应项目不可连接，而不会影响 POS 本机收银。
- [ ] `SimplePay 安全资金模式` 未启用时只显示警告，不会自动启用或发起真实资金操作。
- [ ] Google 管理员在全局总览输入 POS 订单号后，可分别看到 POS、整合任务、SimplePay 和联盟状态。
- [ ] 追踪不存在的订单会提示找不到；追踪操作不会修改任务状态或自动重试。

## 1. 首页和菜单

- [ ] 打开后默认进入 `下单` POS 画面。
- [ ] 左下角菜单按钮可以打开。
- [ ] 左下角菜单顺序为：下单、菜单、库存、转账记录、报告、设置。
- [ ] 点击每个菜单后，页面内容会正确切换。
- [ ] 右下角收银菜单可以打开。
- [ ] 右下角显示已登录操作员、班次状态、员工登录 / 登出。

## 2. 员工登录和班次

- [ ] 联网时点击 `联网 Google 登录`，使用已授权 Google 邮箱登录 POS。
- [ ] 联网时直接提交邮箱密码表单，会提示必须使用 Google 登录。
- [ ] 管理员在 `设置` 里授权 / 更新员工时，需要设置至少 8 位离线密码。
- [ ] 断网时使用授权邮箱 + 离线密码登录 POS。
- [ ] 断网时输入错误离线密码，会拒绝登录。
- [ ] 离线密码登录后恢复网络，系统会保留班次并要求 Google 重新验证。
- [ ] 云端把当前员工设为停用后，恢复网络或下次授权核对会自动退出。
- [ ] 停用员工自动退出后，进行中班次和已完成销售不会丢失。
- [ ] 云端变更员工分行且没有进行中班次时，会切换新分行并清空旧购物车。
- [ ] 云端变更员工分行且仍有班次时，会锁定旧班次并要求管理员交班。
- [ ] 联网登录后员工菜单显示最近一次云端授权核对时间。
- [ ] 管理员 Google 账号直接切换为员工账号后，后台管理员权限立即消失。
- [ ] 管理员登录后默认进入 `全局总览`，菜单只显示 `报告` 与 `设置`。
- [ ] 管理员不能进入下单、菜单、库存或转账记录，也不会自动开始员工班次。
- [ ] 员工登录后菜单显示下单、菜单、库存、转账记录和报告，不显示设置。
- [ ] 员工报告、客户、库存、交班及 CSV 导出只包含授权分行资料。
- [ ] 员工菜单页只能查看商品与售价，不能修改全局商品资料或价格。
- [ ] 联网时提交管理员邮箱密码表单不会直接进入后台，而会要求 Google 登录。
- [ ] Google 管理员可在设置页建立或更新至少 8 位管理员离线密码。
- [ ] 两次管理员新密码不一致时拒绝保存，明文密码不会写入本机或云端。
- [ ] 断网时管理员邮箱错误、未设置密码或密码错误都会拒绝进入后台。
- [ ] 断网时正确邮箱和离线密码可以进入后台。
- [ ] 关闭浏览器后重新打开，管理员和员工都必须重新登录，班次和订单仍保留。
- [ ] 登录后右下角显示当前操作员。
- [ ] 普通收银员登录后，系统自动切换到该员工授权分行。
- [ ] 普通收银员登录后，分行下拉被锁定，不能切换其他分行。
- [ ] 普通收银员可进入 `库存`，但只能调整自己授权分行库存。
- [ ] 普通收银员可进入 `转账记录`，但只能看到和退款 / 作废自己授权分行订单。
- [ ] 在线作废订单后，云端订单状态和商品库存会在同一事务完成。
- [ ] 重复补传或重复执行同一退款不会再次增加库存。
- [ ] 离线作废订单后显示更新待同步，恢复网络后自动完成云端退款事务。
- [ ] 退款时若任一云端商品不存在，不会只回补其他商品，并显示库存待复核。
- [ ] 退款事务模块尚未加载时保持待同步，不会退回整份商品覆盖。
- [ ] 登录后自动开始班次，能看到班次开始时间。
- [ ] 班次进行中退出员工账号，班次资料仍然保留。
- [ ] 原员工重新登录后显示已恢复进行中的班次，班次号和开始时间不变。
- [ ] 进行中班次存在时，其他员工登录会被拒绝，不会覆盖原班次。
- [ ] 进行中班次存在时，分行选择被锁定；管理员也必须先交班才能切换。
- [ ] 管理员登录后可为原员工完成交班，但不能在原员工班次下开始新销售。
- [ ] 管理员代为交班时，交班记录保存实际核对和结班账号。
- [ ] 同分行两个收银台同时营业时，每个班次只统计自己的订单。
- [ ] 开班前可填写备用金；点击“开始收银”后备用金写入当前班次。
- [ ] 开班后可以从右下角登记现金存入或现金取出。
- [ ] 每笔现金流水必须填写正数金额和原因，并记录时间及操作员。
- [ ] 错误流水可以冲正，原流水和反向流水都会保留。
- [ ] 结束班次 / 交班时会出现结算资料。
- [ ] 交班弹窗显示有效订单、销售总额、应有现金、作废订单和付款方式。
- [ ] 应有现金等于备用金加现金销售及现金存入，再减现金取出。
- [ ] 交班弹窗的现金存入和取出来自逐笔流水，不能直接覆盖总数。
- [ ] 现金取出导致应有现金小于零时禁止交班。
- [ ] 输入现金实点后会即时计算现金差额。
- [ ] 现金差额不为零时必须填写交班备注。
- [ ] 有待同步、SimplePay 待确认或联盟待关联时会显示提醒，但确认后仍可交班。
- [ ] 交班前可以导出本班结算 CSV。
- [ ] 本班结算 CSV 包含逐笔现金流水及冲正关系。
- [ ] 到 `设置` 页面可以看到交班记录。
- [ ] 历史交班记录可以分别导出；旧版记录也不会报错。

## 3. 标准收银

- [ ] 选择当前分行。
- [ ] 点击 `简单联盟 RM180 配套` 加入购物车。
- [ ] 商品价格显示 RM 180。
- [ ] 实收金额自动预填为应收金额。
- [ ] 修改数量后，小计、应收、实收建议金额正确变化。
- [ ] 输入折扣后，应收金额和找零正确变化。
- [ ] 填写客户姓名和电话。
- [ ] 可选填联盟推荐码，完成订单后销售记录及 CSV 均保留推荐码。
- [ ] 联盟配套未填写客户电话时不能进入收款；填写电话后即使不填推荐码也会建立联盟任务。
- [ ] Google 管理员点击“同步联盟价格”后，POS联盟商品价格与联盟系统有效配套价格一致。
- [ ] Google 管理员点击“检查整合任务”后，只读取最近50条任务，并显示需要处理的错误。

## 4. SimplePay 和非现金付款

- [ ] 新建一笔订单。
- [ ] 付款方式选择 `简单支付 / SimplePay`。
- [ ] 不填写参考号时，POS 建立待付款订单并显示 SimplePay 二维码。
- [ ] 待付款订单显示“库存已预留”，但不计入销售额、班次收入或付款汇总。
- [ ] 顾客登录 SimplePay 扫码后只能支付二维码内的固定金额。
- [ ] 顾客确认付款后，云端 POS 订单转为正常并写入 SimplePay 参考号。
- [ ] 收据点击“检查付款状态”后，只读取当前订单并显示付款结果。
- [ ] 交易记录的待付款订单也可点击“检查付款状态”。
- [ ] 付款确认后，本机订单变为正常销售并计入报告，不需要整页刷新。
- [ ] 取消待付款订单时，预留库存正确恢复。
- [ ] 取消待付款订单点击按钮并确认即可，不会建立 SimplePay 退款或联盟撤销任务。
- [ ] 订单取消后，顾客再次扫描原付款码会被 SimplePay 拒绝。
- [ ] 已有 SimplePay 付款参考号时，可以填写参考号交由后台验证。
- [ ] 银行转账、DuitNow、TNG 等非现金付款未填参考号时仍会提醒确认。

- [ ] 点击“全部日期”会显示跨日期订单，再点击“今天”恢复当天。
